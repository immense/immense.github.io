php Style Guide

Naming Conventions

Use Pascal case for classes. Camel case for class properties and methods. Underscores for normal variables and static method names.

Spacing and Braces

Set tabs to always be spaces, and use 4 spaces per indent.

Use the Allman style for curly braces for class and function definitions. The curly brace should be on the next line by itself.

Sample

{% highlight php %} 

class TestThing
{
    public $objectProperty;

    public function methodName()
    {
        ...
    } // End methodName

    public static function method_name() 
    {
        ...
    } // End method_name

} // End TestThing

function function_name( $argument = null ) 
{
    ...
} // End function_name

$variable_name = new TestThing(); //Non-object and objects
{% endhighlight %}

Objective-C style guide

Naming Conventions

Use Pascal case for classes. Camel case constants and prefix with the letter k. Underscores for instance variables

Spacing and Braces

Set tabs to always be spaces, and use 4 spaces per indent. (Xcode defaults)

CSS Style Guide

Layout

Use the minimalistic approach with stylesheets. Styles should be on one line, separated by spaces. 


body {font-family: sans-serif; background: #000;}
h1, h2, h3, h4 {font-weight: bold; text-transform: capitalize;}
h1 {font-size: 1.6em;}

General Tips

  • Do not specify a font family for body text. Instead, just use serif or sans-serif.
  • Use inline styles as never as possible.
